---
name: debug
description: Root-cause analysis for bugs, errors, failures, and unexpected behavior
---

- Start from the observed symptom, not assumptions.
- Read stack traces, logs, failing tests, and nearby code paths.
- Form the most likely cause from evidence before touching code.
- Fix the root cause, never patch the symptom.
- Add or suggest regression protection when appropriate.
- Report briefly: cause, fix, and how to verify.
- Do not guess — read the actual code and data.
