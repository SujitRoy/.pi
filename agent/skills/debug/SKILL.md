---
name: debug
description: Investigate bugs, errors, failures, and unexpected behavior with root-cause analysis
---

Use when investigating bugs, errors, failures, or unexpected behavior.

- Start from the observed symptom.
- Read stack traces, logs, failing tests, and nearby code paths.
- Form the most likely cause from evidence.
- Fix the root cause, not only the symptom.
- Add or suggest regression protection when appropriate.
- Report briefly: cause, fix, and how to verify.
