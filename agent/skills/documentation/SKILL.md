---
name: documentation
description: Create and maintain clear, concise, accurate technical documentation
---

- Write for the reader, not for yourself.
- Include code examples for every non-trivial concept.
- Document architecture decisions and trade-offs.
- Keep docs in sync with code — stale docs are worse than no docs.
- Target the audience: developers need different info than operators.
- Prefer examples over abstract descriptions.
- If a feature exists, document how to use it — not why it was built.
