---
name: frontend
description: UI development, accessibility, responsiveness, and client-side best practices
---

- Semantic HTML — use the right element for the job.
- Accessibility is not optional — keyboard nav, screen readers, color contrast.
- Mobile-first responsive design — test on small screens first.
- No inline styles — use CSS classes or design tokens.
- Lazy-load heavy assets — images, fonts, third-party scripts.
- Client-side validation is UX, server-side validation is security — use both.
- Minimize re-renders — memoize expensive computations.
- Bundle size matters — audit and trim dependencies regularly.
